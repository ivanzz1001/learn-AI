# 配置阿里千问大模型

阿里千问3.6Plus大模型有一个免费体验额度，这里我们为OpenClaw装上该大模型来体验一下。

- [阿里千问](https://www.qianwen.com/)


## 1. 配置步骤

>ps: 当前OpenClaw还没有稳定，还在进行高速升级中，建议使用向导来配置模型，而不要直接改配置文件

### 1.1 启动配置向导选择更新已有配置

执行`openclaw onboard`启动配置向导，会提示检测到已存在配置:


![openclaw-inst](https://raw.githubusercontent.com/ivanzz1001/learn-AI/master/openclaw/%E5%AE%89%E8%A3%85openclaw/image/openclaw-qianwen-01.png)

这里我们选择更新已有配置即可。

### 1.2 配置模型提供商

模型提供商选择“千问”：

![openclaw-inst](https://raw.githubusercontent.com/ivanzz1001/learn-AI/master/openclaw/%E5%AE%89%E8%A3%85openclaw/image/openclaw-qianwen-02.png)

然后这里暂时选择“标准API”：

![openclaw-inst](https://raw.githubusercontent.com/ivanzz1001/learn-AI/master/openclaw/%E5%AE%89%E8%A3%85openclaw/image/openclaw-qianwen-03.png)

由于列表中暂时没有最新的`千问3.6Plus`大模型,因此我们选择手动输入模型：

![openclaw-inst](https://raw.githubusercontent.com/ivanzz1001/learn-AI/master/openclaw/%E5%AE%89%E8%A3%85openclaw/image/openclaw-qianwen-04.png)


## 2. 检查模型配置

执行如下命令检查所配置的大模型的状态：

```bash
# openclaw models status
```
![openclaw-inst](https://raw.githubusercontent.com/ivanzz1001/learn-AI/master/openclaw/%E5%AE%89%E8%A3%85openclaw/image/openclaw-qianwen-05.png)

通过上面的输出我们可以得到以下关键信息

<div class="table-wrapper" markdown="block">

|项目              |	状态                                           |说明                               |
|:-----------------|:-------------------------------------------------|:----------------------------------|
|**默认模型**       |`modelstudio/qwen3.6-plus`                        |已经正确设置为阿里云模型             |
|**已配置模型**     | 包含`modelstudio/qwen3.6-plus`                    |模型列表中已有你配置的模型           |
|**Auth 存储位置**  |`~\.openclaw\agents\main\agent\auth-profiles.json` |你的 API Key 就安全地存放在这个文件中|
|**提供商凭证状态**  |`modelstudio: ... api_key=1`                      |存在一个 API Key 类型的凭证         |
|**API Key 预览**   |`sk-63e3c...c4e73944`                             |可以看到部分密钥，证明它已正确加载   |

</div>

你的输出显示`providers w/ OAuth/tokens (0) 和 api_key=1`，这表示 modelstudio 提供商使用了一个 API Key 凭证，而不是 OAuth 令牌。这正是你配置的阿里云 API Key。


## 3. 简单验证

1） 启动openclaw gateway

```bash
# openclaw gateway
```

2) 打开gateway web控制台

```bash
# openclaw dashboard
```
